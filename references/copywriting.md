# 营销文案 -- 多平台营销文案生成

根据产品信息和目标平台特性，运用专业文案框架生成适配的营销文案。

## 能力概览

### 基础能力（独立可用）
- 七大平台文案适配（公众号 / 小红书 / 微博 / 抖音 / 视频号 / 知乎 / LinkedIn）
- 六种专业文案框架（FABE / AIDA / PAS / 倒计时 / STAR / Hook）
- 标题公式库（12 个高转化模板）
- 平台算法适配（各平台推荐机制和限流规则）
- 平台敏感词自查 + 广告合规预检
- 每篇输出 3 个标题方案 + A/B 测试建议
- CTA（行动号召）设计指南

## 连接器（可选增强）

| 连接器 | 增强能力 |
|--------|---------|
| **Notion** | 文案写入 Notion，团队在线协作审阅修改 |
| **Figma** | 引用设计稿辅助文案创作，确保文案与视觉设计一致 |

## 输入要求

**必填**：
1. **产品/活动信息**：名称、核心卖点（至少 1 个）

**选填**（未提供时由 AI 推断或追问）：
2. **目标平台**：微信公众号 / 小红书 / 微博 / 抖音 / 视频号 / 知乎 / LinkedIn / 通用（默认：通用）
3. **文案类型**：品牌推广 / 产品种草 / 活动预热 / 干货分享 / 用户故事
4. **调性要求**：专业严谨 / 轻松活泼 / 走心情感 / 数据驱动
5. **目标受众**：年龄段、职业、兴趣标签
6. **参考竞品**：希望对标或差异化的文案风格

## 执行流程

### 第一步：确认需求与补全信息

**如果用户只给了产品名**：追问"目标平台？想突出哪个卖点？"
**如果信息完整**：直接进入第二步。

### 第二步：选择文案框架

| 类型 | 推荐框架 | 结构 | 适配场景 |
|------|---------|------|---------|
| 产品种草 | FABE法 | Feature特征 → Advantage优势 → Benefit利益 → Evidence证据 | 小红书/公众号 |
| 品牌推广 | AIDA法 | Attention注意 → Interest兴趣 → Desire欲望 → Action行动 | 全平台通用 |
| 活动预热 | 倒计时法 | 悬念铺垫 → 利益预告 → 稀缺感 → 行动号召 | 微博/朋友圈 |
| 干货分享 | PAS法 | Problem问题 → Agitate放大 → Solution方案 | 知乎/公众号 |
| 用户故事 | STAR法 | Situation场景 → Task任务 → Action行动 → Result结果 | 小红书/知乎 |
| 短视频/社交 | Hook法 | 3秒Hook → 痛点共鸣 → 解决方案 → CTA | 抖音/视频号 |

### 第三步：按平台规范生成

#### 微信公众号（深度长文）
- **字数**：1500-3000 字
- **结构**：吸睛标题 → 摘要（50字）→ 开头（场景代入或痛点共鸣）→ 正文（分段+小标题，每段300字内）→ 金句总结 → CTA
- **标题公式**：`数字 + 痛点 + 解决方案` 或 `反常识 + 核心观点`
- **算法要点**：发布后2小时的打开率决定推荐；标题≤22字（超长会被截断）

#### 小红书（种草笔记）
- **字数**：300-800 字
- **结构**：标题（含emoji + 关键词）→ 正文（口语化，短句分段）→ 标签（15-20个）
- **标题公式**：`emoji + 人群 + 痛点 + 数字/解决`
- **算法要点**：收藏率 > 点赞率 > 评论率是推荐权重；首图决定80%点击率
- **注意**：第一句必须是hook，使用第一人称，多用"亲测""真的会谢"等社区用语

#### 微博（短平快传播）
- **字数**：100-280 字
- **结构**：#话题标签# + 开头金句/争议观点 + 正文 + 互动提问
- **算法要点**：前40字决定展开率；带话题+@+投票/图片可获得更多推荐
- **注意**：发布30分钟内的互动量决定后续曝光

#### 抖音（口播脚本）
- **字数**：200-500字（对应60-120秒）
- **结构**：3秒hook → 正文（口语化短句）→ CTA
- **3秒hook公式**：`"千万别XX" / "99%的人不知道" / "我花了X万才发现"`
- **算法要点**：完播率 > 互动率 > 转发率；前3秒决定是否被推荐进下一个流量池

#### 视频号（中视频/短视频）
- **字数**：口播稿300-800字（对应2-5分钟）
- **结构**：问题引入 → 干货内容（3-5个要点）→ 总结金句 → 引导关注
- **算法要点**：社交推荐（朋友点赞可见）权重极高；评论区互动比抖音更重要
- **注意**：开头不要自我介绍，直接抛出价值点

#### 知乎（专业回答/文章）
- **字数**：800-3000 字
- **结构**：一句话结论开头 → 论据展开（数据/案例/逻辑推导）→ 补充说明 → 利益声明
- **算法要点**：「专业度」权重高——引用数据和来源可提升推荐；首赞和首评时效性很重要
- **注意**：避免明显广告感，强调专业性和数据支撑

#### LinkedIn（职业化内容）
- **字数**：500-1500字（英文或中英双语）
- **结构**：一句话洞见 → 个人经历/数据支撑 → 行业观点 → 开放式提问
- **注意**：避免过度营销，强调价值分享和行业思考

### 第四步：标题优化

#### 标题公式库

| 编号 | 公式 | 示例 | 适配平台 |
|-----|------|------|---------|
| 1 | 数字 + 痛点 + 解决 | 《月销10万单背后：3个被忽略的运营细节》 | 公众号/知乎 |
| 2 | emoji + 人群 + 发现 | 敏感肌姐妹看过来！这个成分真的救了我 | 小红书 |
| 3 | 反常识 + 真相 | 《为什么你的SEO越做越差？90%的人踩了这个坑》 | 公众号/知乎 |
| 4 | 禁止 + 好奇 | "千万别这样护肤！我踩坑3年才明白" | 抖音/视频号 |
| 5 | 对比 + 选择 | 《A和B到底选哪个？真实使用30天告诉你答案》 | 知乎/小红书 |
| 6 | 清单 + 收藏 | 2024年必备的10个效率工具（建议收藏） | 小红书/公众号 |
| 7 | 身份认同 | "月薪3万的人都在用的XX方法" | 抖音/小红书 |
| 8 | 时效紧迫 | 《还不知道就亏了！XX限时免费中》 | 微博/朋友圈 |
| 9 | 结果前置 | "用了这个方法，3天涨粉5000" | 抖音/视频号 |
| 10 | 争议引导 | "月薪5千和5万的人，最大区别是什么？" | 知乎/微博 |
| 11 | 避坑指南 | 新手千万别踩的5个坑（附正确做法） | 小红书/知乎 |
| 12 | 权威背书 | 被XX推荐过的宝藏产品，终于等到降价了 | 小红书/微博 |

每篇文案输出 3 个标题方案，并给出 A/B 测试建议（推荐哪2个做测试，预判各自优势）。

### 第五步：CTA（行动号召）设计

| CTA类型 | 适用场景 | 话术示例 | 心理机制 |
|--------|---------|---------|---------|
| 低门槛行动 | 首次触达 | "点击免费试用""领取资料包" | 降低决策成本 |
| 紧迫感驱动 | 促销/限时 | "最后3小时""仅限前100名" | 损失厌恶 |
| 社交证明 | 种草/口碑 | "已有10万人选择""看看大家怎么说" | 从众心理 |
| 利益承诺 | 转化页 | "30天无理由退款""不满意全额退" | 消除顾虑 |
| 互动引导 | 社交平台 | "你觉得呢？评论区聊聊""收藏=不迷路" | 参与感 |

### 第六步：检查与优化

完成文案后执行检查清单：
- [ ] 标题是否有吸引力？是否有3个备选？
- [ ] 开头3秒/首屏是否有强hook？
- [ ] CTA 是否明确且匹配平台习惯？
- [ ] 是否包含《广告法》禁止的极限词？如有，立即替换
- [ ] 内容是否符合平台调性和字数要求？
- [ ] 标签/话题是否充足且相关？
- [ ] 平台敏感词自查：
 - 小红书：避免"购买链接""点击下方""私信""微信"等引流词
 - 抖音：避免"直播间""秒杀""最低价"等限流词
 - 微博：避免敏感政治词和低俗词汇

## 输出格式

```
## [平台名] 文案

### 标题方案（3选1）
1. [标题A — 风格说明]
2. [标题B — 风格说明]
3. [标题C — 风格说明]

**A/B测试建议**：推荐标题A和标题B做测试。A偏[数字型]适合追求点击率；B偏[情感型]适合追求分享率。

### 正文
[按平台结构要求的完整文案]

### 标签/话题
[平台对应的标签列表]

### 文案策略说明
- 框架：[使用了哪个框架及选择原因]
- 核心卖点：[突出了哪个卖点及原因]
- 目标情绪：[想激发用户什么情绪/行动]
- CTA策略：[使用了哪种CTA及心理机制]
- 发布建议：[最佳发布时间 + 配图/封面建议]
```

如用户指定多个平台，为每个平台分别生成完整文案（切入角度须不同）。

**示例（小红书 — 新品发布）**：


## 质量标准

- 同一产品不同平台的文案不能简单改字数，必须重新构思切入角度和表达方式
- 每篇文案必须有明确CTA，且CTA形式匹配平台习惯
- 标题必须提供3个方案 + A/B测试建议
- 广告合规预检：自动排查极限词，发现后在文案中直接替换并标注
- 平台敏感词自查：避免触发限流机制
- 文案必须完整可用，非半成品或框架大纲

## 场景变体

### 新品发布 — 重点突出"新"的价值，制造期待感，使用倒计时法
### 日常种草 — 重点突出真实体验，使用FABE法，多用口语
### 促销活动 — 重点突出稀缺性和紧迫感，强化CTA
### 品牌故事 — 重点突出品牌理念和情感连接，使用STAR法
### 批量套装 — 用户给一个产品，一次性生成3-5个平台的配套文案（各自独立调性和切入角度）

## 输入不足处理

| 缺失项 | 处理方式 |
|--------|---------|
| 产品/活动信息 | 必填项，缺失时向用户索取，不得仅凭名称虚构产品卖点 |
| 目标平台 | 默认使用"通用"格式，标注"未指定平台，已按通用格式生成" |
| 文案类型 | 根据产品信息推断类型，标注推断依据 |
| 调性要求 | 结合文案类型和平台特性选择默认调性，标注"默认调性，可按需调整" |
| 目标受众 | 标注"未指定受众，已按产品一般受众生成" |
| 参考竞品 | 缺失时不做竞品对标分析 |

## 关联Skill

- **品牌调性审查** — 文案生成后由本技能审查品牌调性一致性
- **广告合规检查** — 文案生成后由本技能做广告法合规预检
- **活动策划** — 活动方案确定后由本技能生成各渠道传播文案
- **SEO内容优化** — SEO优化内容由本技能做平台适配和文案润色
- **社媒热点追踪** — 热点选题确定后由本技能生成借势文案
- **竞品追踪** — 竞品文案策略分析指导自身文案差异化方向
- **营销效果分析** — 文案A/B测试效果数据由本技能分析优化方向

## 失败回退机制

| 步骤 | 失败条件 | 回退目标 | 最大重试 | 不可恢复时升级路径 |
|------|---------|---------|---------|-----------------|
| 平台识别 | 用户未指定平台或平台需求模糊 | 默认输出"通用版"文案 + "小红书版"文案作为对比，提示用户指定平台可获得更精准适配 | 1次 | 输出通用版，附注各平台差异要点供用户自行适配 |
| 文案框架选择 | 产品特性与框架不匹配（如PAS适合痛点型产品但不适合品牌型） | 输出2-3个不同框架的标题方案，标注各框架适用场景，由用户选择 | 2次 | 默认使用AIDA框架（通用性最强），标注"可换用其他框架优化" |
| 敏感词检测 | 检测到疑似极限词但上下文可能为合理描述 | 标注疑似项并给出替换方案，同时保留原文供用户判断 | 1次 | 含"最""第一"等词但在引用场景（如"行业第一份报告"）→ 标注建议核实 |
| 调性把握 | 多次生成后用户对调性仍不满意 | 提供3个调性方向示例（正式/亲切/幽默），请用户选择偏好方向后重新生成 | 2次 | 建议用户提供参考文案链接，Agent基于参考反推调性参数 |

## 红线规则

1. **广告法合规**：必须排查极限词（"最""第一""国家级""唯一""顶级"等），发现后通知用户并给出替换建议
2. **不得虚构数据**：不得编造"月销10万单""10万人选择"等无来源的营销数据
3. **平台合规**：不得建议涉及平台禁止的引流行为（如小红书留微信号、抖音引导私下交易）
4. **内容真实性**：不得夸大产品功效或做虚假承诺，文案中对功效的描述需标注"需用户自行验证"
5. **多平台差异化**：同一产品不同平台的文案必须重新构思切入角度，严禁简单改字数敷衍

## 使用建议

- 小红书标题带 emoji 点击率平均高 15-20%
- 微博前 40 字是"展开全文"的生死线，必须在此抛出hook
- 知乎开头直接给结论再展开论证，符合平台用户阅读习惯
- 视频号重视评论区运营，文案中可预埋互动话题
- 抖音前3秒完播率决定流量，hook必须在第一句话
